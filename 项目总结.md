# 记忆典当行 — 项目总结

## 项目简介

**记忆典当行** 是一款赛博朋克风格的叙事冒险游戏，以跨平台小程序（微信小程序 + H5）形式呈现。玩家扮演一家记忆当铺的老板，通过接待顾客、鉴定记忆、谈判交易、管理库存，体验关于人性与记忆的黑暗故事。

项目由 **Coze 编程 CLI** 脚手架生成，采用 **pnpm monorepo** 架构，包含前端（Taro + React）和后端（NestJS）两个包。

## 核心文件说明

### 入口文件

| 文件 | 作用 |
|------|------|
| `src/app.tsx` | 根 React 组件，初始化图标上下文、框架预设层、全局 Toast |
| `src/pages/index/index.tsx` | 唯一页面，根据游戏阶段路由到 6 个画面之一 |
| `server/src/main.ts` | NestJS 后端入口，监听 3000 端口 |

### 游戏核心

| 组件/文件 | 功能 |
|-----------|------|
| `hub-screen.tsx` | **当铺主界面** — 展示资金/声望/库存、负能量计量、氛围描述、行动按钮 |
| `dialog-screen.tsx` | **对话界面** — 顾客头像、心理状态条、打字机效果对话、玩家选项 |
| `appraisal-screen.tsx` | **记忆鉴定** — 频率调谐（滑块匹配）+ 杂质去除（点击清除） |
| `trading-screen.tsx` | **交易谈判** — 购买/拒绝/敲诈/篡改四种行动，不同风险收益 |
| `result-screen.tsx` | **结果展示** — 交易结局、后果、记忆核心故事 |
| `storage-screen.tsx` | **记忆仓库** — 库存列表、纯度/完整度/腐化度条、记忆冲突、上架出售 |

### 数据与状态

| 文件 | 内容 |
|------|------|
| `game-store.ts` | Zustand store：游戏阶段、天数、资金、声望、顾客队列、对话进度、鉴定状态、库存、操作日志等 |
| `game-data.ts` | 静态数据：顾客档案（含完整对话树）、记忆定义、背景故事。当前有完整实现的顾客"老陈" |

## 游戏机制

- **六阶段循环**：当铺大厅 → 顾客对话 → 记忆鉴定 → 交易谈判 → 结果展示 → 记忆仓库
- **心理状态系统**：每位顾客有"防御值"和"紧迫值"，玩家对话选择会影响这些数值
- **鉴定小游戏**：频率调谐（拖拽滑块匹配情感频率）+ 杂质去除（点击记忆球上的黑斑）
- **四种交易行为**：购买收购 / 拒绝交易 / 敲诈勒索 / 篡改记忆（各有风险与后果）
- **记忆库存管理**：记忆会随时间腐化、相邻冲突记忆产生负面效果、可用固定剂或出售处理

## 运行方式

```bash
# 安装依赖
pnpm install

# 开发模式（H5 + 后端并发启动）
pnpm dev          # H5 端口 5000，后端端口 3000

# 构建
pnpm build:web    # H5 构建
pnpm build:weapp  # 微信小程序构建
pnpm build:server # 后端构建
```
